# Features Directory

This directory contains feature-specific code organized by domain.

## Structure

- `auth/` - Authentication and user management
- `instruments/` - Instrument master data management
- `people/` - People master data management
- `locations/` - Location master data management
- `movements/` - Checkout and return operations
- `maintenance/` - Maintenance logging and tracking
- `usage/` - Usage event logging
- `analytics/` - Analytics, reports, and dashboards
- `admin/` - Admin panel (users, roles, permissions)

## Guidelines

Each feature directory should contain:
- Components specific to that feature
- Hooks for feature-specific logic
- Types/interfaces for the feature
- Services/API calls for the feature
